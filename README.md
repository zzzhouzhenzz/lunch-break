# lunch-break

A tiny Claude Code plugin for stepping away from long sessions without losing your place.

After a long stretch of work — especially across multiple cc sessions — coming back from a break can feel like waking up in someone else's codebase. Resuming the cc session reloads the context window, but *you* already forgot everything. This plugin writes a short, durable handoff note to disk so future-you can catch up in 30 seconds.

## Commands

### Before you step away

```
/lunch-break            # save current session context
/lunch                  # alias
/break                  # alias
/lunchbreak             # alias
```

Claude reflects on the session, writes a markdown handoff to `~/.claude/lunch-break/YYYY-MM-DD-HHMM-<slug>.md`, and generates a short descriptive title. You can optionally append a note: `/lunch-break waiting on PR review`.

### When you come back

```
/lunch-back             # list saved sessions, pick one
/lunch-break-back       # alias
/back                   # alias

/lunch-back 1           # load the most recent
/lunch-back fix-cookie  # load by slug / partial title
```

Claude lists saved sessions newest-first with titles and ages. Pick one and it walks you through: goal, done, ongoing, todo — and tells you the single next action to take.

## Saved file format

```markdown
---
title: Lunch-break plugin build
slug: lunch-break-plugin-build
saved_at: 2026-04-12T16:02:00-07:00
cwd: /Users/you/code/ml
branch: main
---

# Lunch-break plugin build

## Goal
...

## Done
- ...

## Ongoing
- ...

## Todo
- ...

## Next
<the single most important thing to do first>

## Notes
...
```

## Install

From a local marketplace:

```
/plugin marketplace add /path/to/lunch-break
/plugin install lunch-break@lunch-break-marketplace
```

## Design notes

- Pure slash commands. No hooks, no MCP server, no background processes.
- Storage is global (`~/.claude/lunch-break/`) so handoffs work across projects and worktrees.
- Title is generated by the model at save time, so you see a useful name in the list instead of a timestamp.
